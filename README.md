LLM+RAG ассистент для работы в монорепозитории

Платформа для совместной работы над проектами использует крупный монорепозиторий, в котором хранится исходный код, документация и конфиги. Пользователи хотят как можно быстрее находить нужную информацию, понимать устройство системы и вносить изменения с меньшим риском. Для этого внедряется LLM+RAG-ассистент, работающий поверх монорепозитория.

Пользователи: команда из начинающих и опытных разработчиков, инженеры по тестированию, специалисты по эксплуатации и сопровождению, технические писатели и аналитики, продуктовые и проектные менеджеры.

Требования:  
- пользователи задают ассистенту вопросы на естественном языке и получают ответы с точными ссылками на файлы, директории и фрагменты кода, а также выдержки из документации;
- по тексту задачи ассистент помогает найти модули и файлы, в которых с наибольшей вероятностью нужно внести изменения;
- по выбранному файлу, модулю или директории ассистент объясняет их назначение, основные зависимости и роль в системе;
- ассистент умеет искать по монорепозиторию в терминах предметной области, а не только по именам файлов и символов;
- ассистент показывает возможное влияние изменений: по файлу или коммиту подсвечивает связанные модули, сервисы и части функциональности, на которые изменение может повлиять;
- ассистент отвечает на вопросы о конфигурации окружений, CI/CD-пайплайнах и инфраструктурных файлах;
- ассистент учитывает права доступа: пользователю показываются только те артефакты монорепозитория, к которым у него есть разрешения, при этом все запросы и ответы логируются;
- ассистент доступен через веб-интерфейс (чат) или через расширение браузера/IDE.

Дополнительный контекст:
Пользователи привыкли к ИИ-ассистентам и ожидают быстрое получение ответов по проекту без долгого ручного поиска. Объём монорепозитория постоянно растёт, документация быстро устаревает, а знание архитектуры и исторических решений сосредоточено у ограниченного числа людей. Так же зачастую у новых сотрудников проблемы с тем, чтобы оперативно включиться в процесс разработки. Внедрение LLM+RAG-ассистента рассматривается как часть долгосрочной стратегии по улучшению опыта работы пользователей с системой и модернизации процессов разработки и сопровождения.
